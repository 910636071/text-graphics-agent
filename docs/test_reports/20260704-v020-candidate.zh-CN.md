### 贪吃蛇 Proposal Smoke

任务：为 `agent_workspace/snake_game.html` 生成单文件浏览器贪吃蛇 proposal。

proposal 审查观察结果：

```json
{
  "live_calls": 1,
  "parse_ok": true,
  "accepted": true,
  "violations": [],
  "used_repair": false,
  "evidence": ["agent_workspace/snake_game.html"],
  "proposed_scopes": ["agent_workspace/snake_game.html"],
  "output_chars": 2128,
  "output_has_html": true,
  "output_has_script": true,
  "output_has_canvas": true,
  "output_mentions_snake": true
}
```

边界：这不是可玩游戏验收。它只证明真实模型能产出可解析、并通过当前
`AgentProposal` 和 `ConstraintChecker` 闸门的 proposal。文件没有写入，浏览器没有运行，蛇移动、碰撞、得分、重开和布局都没有做功能验收。完整功能测试要等 G3 加上写工具 staging 和 accepted-record commit 语义后才能成立。

### 本地贪吃蛇产物功能检查

在 proposal smoke 边界澄清之后，Codex 本地写入了
`examples/snake_game.html`，并用 headless Chromium 做功能验收。这是本地产物验收，不是 TGA 子 agent 已经具备写文件能力的证据。

观察结果：

```json
{
  "ok": true,
  "title": "TGA Snake",
  "canvas": {
    "width": 520,
    "height": 520
  },
  "nonBlankPixels": 230400,
  "scoreAfterEat": 1,
  "lengthAfterEat": 4,
  "selfCollisionGameOver": true,
  "wallCollisionGameOver": true,
  "restartScore": 0,
  "pauseToggle": ["Paused", "Running"]
}
```

已验收项目：

- 页面可以作为独立浏览器文件打开
- canvas 渲染非空游戏内容
- 吃到食物后分数增加，蛇身变长
- 自撞后进入 Game Over
- 撞墙后进入 Game Over
- Restart 清除 `gameOver` 并重置分数
- Pause/Resume 按钮能更新状态
- 桌面和移动端布局都没有横向溢出

## 剩余发布边界

这个候选版本支持 G1 和 G2 的本地主张：

- 文件派生 evidence provenance 可审计
- deterministic clean acceptance metrics 和 false-positive reporting
- live proposal smoke 与 live chat smoke，其中 proposal smoke 明确不等于功能验收
- 本地贪吃蛇产物功能检查，并且明确不等于 TGA 写能力子 agent 主张

它还不应声称已经具备写能力 agent 执行。下一个 release gate 是 G3：写工具必须先 staging，并且只能在 accepted records 之后 commit effects。
