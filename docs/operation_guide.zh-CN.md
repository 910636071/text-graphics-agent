# 操作指南

本指南面向本地 Web 客户端：`http://127.0.0.1:8012/`。

## 第一次启动

1. 启动客户端：

   ```powershell
   python -m text_graphics_agent.gui
   ```

2. 打开 `http://127.0.0.1:8012/`。
3. 需要英文界面时，点击右上角 `EN` / `中` 语言切换按钮。
4. 应用默认进入**聊天流**——像 ChatGPT 一样自然输入即可。

## 界面概览

- **侧边栏**："新对话"按钮、对话搜索框、历史对话列表（持久化到 localStorage）、导航区（工作台、审批、自动化、设置、指南、诊断工具）。
- **聊天流**：主区域。闲聊直接回复；任务请求（"检查设置面板布局"）通过安全管道派发，结果以卡片形式返回。
- **输入区**：底部输入框。Enter 发送（Shift+Enter 换行）。它只保留需求文本、示例任务、模型配置、Live LLM 开关和一个精简的任务范围摘要。
- **任务范围面板**：聊天旁边默认折叠的右侧卡片。可以设置本次文件范围、可选验收锚点，从工作区浏览文件，粘贴路径，或拖入文件，不需要离开对话。
- **Inspector**（用 👁 按钮切换）：右栏面板，显示任务范围、任务上下文、权限边界和策展记忆（标注为不可信）。
- **顶栏**：标题、语言切换、Inspector 切换、快捷操作（指南、健康检查、自动化、设置）。

## 核心使用流程

1. 在输入框输入消息，按 Enter 发送。
2. 如果是闲聊（"你好"、"你觉得怎么样"），TGA 直接回复，不派发子 Agent。
3. 如果是任务（"检查设置面板布局并给出修改方案"），TGA 会：
   - 分解意图并检测污染（意图防火墙）
   - 将请求净化为 `TaskSpec`（原始用户文本不会到达子 Agent）
   - 通过 `AgentRegistry` 路由到最匹配的子 Agent
   - 子 Agent 生成 `AgentProposal`
   - 17 条约束检查审查提案
   - 结果以卡片形式出现在聊天流中，显示接受/拒绝状态和违规详情
4. 如果提案被拒绝，聊天卡片会显示每种违规类型的具体修改建议。
5. 点击任务结果卡片上的"查看详情"可查看完整工作流时间线和审计信息。

## 对话历史

- 所有对话自动保存到 `localStorage`。
- 侧边栏显示历史列表，标题从首条消息自动生成。
- 使用搜索框按标题或内容过滤对话。
- 点击对话加载内容；hover 显示删除按钮。
- 刷新页面后对话仍在。

## 设置

从侧边栏或顶栏打开**设置**。三个分区：

1. **连接**：选择提供商（DeepSeek、OpenAI、Gemini、Mock），输入 API Key，可选模型名称。点击"测试连接"验证。
2. **文件范围默认值**：设置任务没有单独指定范围时使用的兜底文件路径。日常任务通常应该在聊天旁边的**任务范围**面板里设置本次范围。
3. **安全规则**：开关 17 条约束检查中的任意一条（高风险变更需要审批）。

## 聊天旁边的任务范围

右侧**任务范围**卡片是发送任务前收窄工作边界的主要入口。它默认折叠；点击输入区的**文件**摘要，或把文件拖到输入区时会展开。

1. 在**本次文件**中添加文件路径。可以用预设（`play.html`、`docs/`、`*.py`），也可以点**浏览文件**从工作区选择。
2. 可以直接粘贴工作区相对路径。如果粘贴的是包含 `text-graphics-agent` 的本机绝对路径，客户端会自动裁成工作区相对路径。
3. 可以把文件拖到输入区或任务范围卡片。浏览器不能可靠暴露本机绝对路径，所以客户端会尽量用文件名匹配工作区文件列表。
4. 在**验收锚点**里添加短锚点，例如 `settings panel` 或 `NPC dialogue`，用于要求结果证明特定证据链。
5. 输入区摘要会显示当前任务使用默认范围，还是本次范围。
6. 本次文件范围和锚点会随当前会话自动保存，并且只覆盖当前提交，不会改写设置页默认值。

## 策展记忆

母 Agent 跨会话积累记忆——常用文件范围、任务模式、违规反馈。

- 记忆是**不可信上下文**：帮助母 Agent 推理，但绝不进入 `TaskSpec.objective`，绝不影响约束裁决。
- 在 **Inspector** 面板（用 👁 按钮打开）的"母Agent记忆"区域查看。
- 每条记忆显示类别、置信度百分比和内容。
- 点击 × 删除单条记忆。
- 记忆随时间衰减（每天 5%），置信度低于 15% 自动剪枝。

## Live LLM

1. 在**设置 → 连接**中配置提供商和 API Key。
2. 测试连接。
3. 在输入区勾选 **Live LLM** 复选框。
4. 提交任务。首次 Live 调用会触发审批检查点。
5. 子 Agent 调用真实 LLM，解析响应，如果首次提案未通过 precheck 则自动修复。

## 诊断工具

侧边栏的诊断区用于验证平台边界，不是日常入口：

- **运行示例**：演示派发 + 约束管道。
- **健康检查**：验证平台配置。
- **基准对照**：运行 11 场景确定性基准测试。
- **安全实验室**：交互式对抗场景测试。
- **诊断搜索**：搜索对抗场景库。

## 安全模型

这个 UI 只是操作面板，不是最终权限层。子 Agent 只能提出提案，约束检查器和人类审批环决定是否继续。策展记忆是不可信的，不能影响约束裁决。
